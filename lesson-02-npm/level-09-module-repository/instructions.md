# Module Repository

## Objective

Explore the GitHub repository of an NPM package.

## Benefits

Many NPM packages have a GitHub repository that displays details from a `README.md` file.

## Complete these tasks

1. In the NPMJS website, find and select `figlet`.
2. In the details page, click on the GitHub link for `figlet`.
3. Observe that the source code and `README.md` are displayed.
4. In a VS Code terminal, navigate to this level folder.
5. Use `npm` to initialize a new project.
6. Install `figlet`.
7. In `script.js`, use `figlet` to display a message.
8. Let the message be `Hello Figlet!`.
9. Run `script.js`. You may need to fix a module error.

## More Information

- The GitHub repository may show different details than the NPMJS page of a package.
- The NPMJS page is for installing a package.
- The GitHub repository is for cloning and modifying a package.

## Usage Tips

- `README.md` files usually contain details on how to install and use code.
- NPM packages are always downloaded from NPM, not from GitHub.
- NPM and GitHub are not always synced, so it's possible the code from NPM is different than code in GitHub.

## Hints

- The NPMJS page shows how often a package is installed.
- The GitHub repository shows how many stars a project has received.
